IPOS ↔ Anybus M40 Variable Map

This document describes the Profinet variable interface implemented in the IPOS Anybus M40 firmware.
It defines how status, input, and output data are exchanged between:

  • STM32 Master (Housekeeping Board)
  • Anybus M40 Module
  • Profinet PLC

1. Overview

The M40 firmware extends the standard HMS application to expose a set of custom process-data variables over the Profinet bus.
The STM32 master communicates with the M40 through UART 3, using a lightweight 16-bit variable protocol.
Each variable corresponds to a register in the Anybus Object Dictionary and is updated cyclically (~50 ms).

2. Communication Flow

PLC ⇄ Profinet ⇄ Anybus M40 ⇄ UART3 ⇄ STM32 Master
| | |
| | ├── Reads/Writes PORTA/B/C
| └── Maps variables into OD (VAR_xx)
└── Reads STATUS and drives control bits

3. Process-Data Variables

Variable ID Symbol Name Direction Bit-Width Description
0x200 VAR_STATUS_PLC Input (M40 → PLC) 16 bit Status word received from PLC via Profinet input area. Bit 0 = Reset Latches pulse; Bit 1 = Force Error.
0x201 VAR_PORTA_OUT Output (PLC → M40) 16 bit Digital outputs (PORTA) set by PLC; relayed to STM32.
0x202 VAR_PORTB_OUT Output (PLC → M40) 16 bit Digital outputs (PORTB) set by PLC; 10 bits used.
0x203 VAR_PORTC_IN Input (M40 → PLC) 16 bit Input snapshot from STM32 PORTC. Updated cyclically.
0x204 VAR_STATUS_DEBUG Input (M40 → PLC) 16 bit Diagnostic flags and system status bits for debugging.

4. Bit-Field Definitions

VAR_STATUS_PLC (0x200) — PLC → STM32

Bit Symbol Description
0 RESET_LATCHES Pulse high to reset safety latches on STM32 (master acknowledges).
1 FORCE_LATCH_ERROR Simulates a fault condition for testing.
2 ENABLE_OUTPUTS Allow relay/IO activation after startup.
3 CLEAR_LOGS Clears non-volatile event log.
15 .. 4 — Reserved for future use.

VAR_STATUS_DEBUG (0x204) — STM32 → PLC

Bit Symbol Description
0 SYS_READY Firmware ready / all tasks running.
1 THERMO_FAULT Thermocouple error detected.
2 RELAY_FAULT Relay state mismatch.
3 WATCHDOG Watchdog event triggered since last reset.
4 LATCH_FAULT Latch hardware fault latched.
5 USB_ACTIVE USB logging task active.
6 PLC_ONLINE Profinet link established.
7 DEBUG_VERBOSE Verbose USB debug enabled.
15 .. 8 — Reserved for future diagnostics.

5. UART Link Frame Format

Field Size Description
Header 1 B 0xAA start byte
Var ID 1 B Low-byte of variable index (0x00–0xFF)
Value 2 B Little-endian 16-bit data
CRC 1 B XOR of previous bytes

Example – Write PORTA = 0x1234
→ AA 01 34 12 9F

6. Profinet Object Dictionary Mapping

Index (hex) Name Object Type Direction Size Comment
0x3000 IPOS_Status Process Object IN 2 bytes Mapped to VAR_STATUS_DEBUG
0x3001 PortC_Inputs Process Object IN 2 bytes Mapped to VAR_PORTC_IN
0x3002 PortA_Outputs Process Object OUT 2 bytes Mapped to VAR_PORTA_OUT
0x3003 PortB_Outputs Process Object OUT 2 bytes Mapped to VAR_PORTB_OUT
0x3004 PLC_Control Process Object OUT 2 bytes Mapped to VAR_STATUS_PLC

7. Timing and Synchronization

  • UART update period ≈ 50 ms (default)
  • Profinet I/O cycle time 1–4 ms (configurable)
  • Each M40 update cycles through all variables:
    1. Read incoming PLC outputs
    2. Forward to STM32 via UART3
    3. Read STM32 inputs and status
    4. Update Profinet input area
